Big Data Analytics

Research: "Mengatasi Tantangan Pengelolaan Obat Safe Medicare Berbasis BigQuery Machine Learning"

Abstrak: Penelitian ini membahas bagaimana saya menerapkan kerangka kerja analitik prediktif menggunakan BigQuery Machine Learning (BQML) untuk mengatasi masalah manajemen inventaris di Safe Medicare[cite: 67]. Tujuan utamanya adalah mengoptimalkan ketersediaan stok obat dan meningkatkan kepuasan pelanggan melalui pendekatan berbasis data[cite: 69].

1. Latar Belakang Masalah

Safe Medicare menghadapi tantangan besar berupa ketidakseimbangan antara ketersediaan stok dan permintaan pasien[cite: 79]. Sistem yang terfragmentasi menyebabkan inefisiensi, seperti kelangkaan obat flu saat musim hujan atau penumpukan stok obat yang kurang populer[cite: 81, 82]. Hal ini tidak hanya membebani finansial perusahaan tetapi juga mengancam reputasi layanan kesehatan[cite: 83, 85].

2. Metodologi

Dalam studi ini, saya melakukan proses pengolahan data yang komprehensif, meliputi:

  • Data Preparation: Pembersihan data (handling null values) dan penghapusan data bias[cite: 201, 212].
  • Feature Engineering: Menambahkan fitur seperti month_number untuk pola musiman dan mengelompokkan waktu tunggu menjadi kategori (Cepat, Normal, Lama)[cite: 227, 233].
  • Tools: Menggunakan teknologi BigQuery Machine Learning untuk memproses volume data besar secara real-time tanpa memindahkan data dari warehouse[cite: 89].

3. Model dan Analisis

  • Linear Regression (Peramalan Permintaan): Saya menggunakan model ini untuk memprediksi volume penjualan obat di masa depan. Model ini memperhitungkan tren musiman, promosi, dan harga[cite: 70, 297].
  • Logistic Regression (Klasifikasi Kepuasan): Model ini digunakan untuk memprediksi apakah pelanggan akan puas atau tidak, berdasarkan variabel seperti waktu tunggu dan ketersediaan obat[cite: 70, 343].

4. Hasil Evaluasi

Hasil evaluasi model menunjukkan performa yang sangat luar biasa:

  • Model prediksi permintaan mencapai nilai RÂ² sebesar 0.997 (mendekati sempurna) dengan Mean Absolute Error (MAE) yang rendah[cite: 71, 354].
  • Model klasifikasi kepuasan pelanggan mencapai akurasi 100% (1.0), membuktikan bahwa waktu tunggu di atas 60 menit adalah faktor kritis penyebab ketidakpuasan[cite: 71, 348].

5. Implementasi dan Kesimpulan

Implementasi solusi ini berhasil mengubah pola kerja dari reaktif menjadi proaktif. Saya berhasil membantu mengurangi beban kerja manual staf dari 15 jam menjadi hanya 4 jam per minggu dan mengurangi risiko limbah obat kadaluwarsa sebesar 65,4%[cite: 73, 354]. Sistem ini memastikan ketersediaan obat bagi pasien secara berkelanjutan.
